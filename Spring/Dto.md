## DTO란?

**정의**
- Data Transfer object의 약자
- 프로세스 간에 데이터를 전달해주는 객체 역할


**나만의 해석**
- 데이터 전송 객체로 어떤 데이터를 담고있는 객체이며 해당 데이터를 다른 곳에서 참조해야할 때 DTO에 담아서 보내준다.
- 데이터 전달 = 데이터 참조 또는 파라메터로 받기 등으로 해석했다.

## dto를 쓰는 이유

- 프로세스 간 메소드 호출의 횟수를 줄이기 위해 사용한다.

    외부 통신 프로그램에서 호출은 큰 비용이므로 이를 줄여야 한다.

    이를 효율적으로 처리하기 위함

- 효율적으로 처리하기 위해 데이터를 모아 한번에 전달하는 방법이 고안됨.


## 나만의 예시

게시판 시스템에서 브라우저는 id, 제목, 내용 값을 사용자에게 받는다.

이 데이터를 컨트롤러에 전송하여 로직 처리를 할려고 한다.

만약 브라우저에서 id, 제목, 내용을 하나하나씩 보내면 3번의 호출이 발생하게 된다.

근데 이거를 dto에 모아서 dto 하나만 전달하면 한 번의 호출만 발생함.
